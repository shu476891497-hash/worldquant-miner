## 1. 项目背景与原版对比

### 原版 worldquant-miner（Generation One）
- 基于预设模板，随机组合因子表达式
- 无持久化记忆，每次重启从零开始
- 无 AI 生成，单纯模板填空
- 数据字段（datafield）列表为手动维护的小列表
- 因子相关性无过滤，容易 SELF_CORRELATION 失败

### Generation Two（本版本）升级点

| 能力 | 原版 | 本版 |
|---|---|---|
| 因子生成方式 | 随机模板填空 | 遗传算法 + 本地 AI（Ollama）主题生成 |
| 数据字段数量 | 手动维护约 20 个 | **自动 API 爬虫，全量 2663 个** |
| 历史记忆 | 无 | SQLite 持久化，跨重启保留精英因子 |
| 相关性过滤 | 无 | CorrelationTracker 预过滤高相关候选 |
| 语法验证 | 无 | TemplateValidator 96KB AST 检查器 |
| 结果可见性 | 只看日志 | `discovered_alphas.txt` 自动记录优质因子 |
| GUI 监控 | 无 | Tkinter GUI 实时监控仪表盘 |

## 2. 系统整体架构

```
┌─────────────────────────────────────────────────────────────────┐
│                    continuous_evolution.py                       │
│                       (主控制器/入口)                            │
│                                                                  │
│   ┌──────────┐  ┌──────────────┐  ┌─────────────────────────┐  │
│   │ 遗传算法  │  │  Ollama AI   │  │   BacktestStorage       │  │
│   │ 变异/交叉 │  │  主题驱动生成 │  │   SQLite 持久化记忆     │  │
│   └────┬─────┘  └──────┬───────┘  └──────────┬──────────────┘  │
│        │               │                      │                 │
│   ┌────▼───────────────▼──────┐   ┌───────────▼──────────────┐  │
│   │   TemplateValidator       │   │   CorrelationTracker     │  │
│   │   语法/AST 预检            │   │   相关性预过滤            │  │
│   └────────────┬──────────────┘   └───────────┬──────────────┘  │
│                │                              │                 │
│   ┌────────────▼──────────────────────────────▼──────────────┐  │
│   │               SimulatorTester                            │  │
│   │        提交因子 → WQ Brain API → 等待结果                 │  │
│   └──────────────────────────────────────────────────────────┘  │
└─────────────────────────────────────────────────────────────────┘
```

**数据流向**：
```
2663个字段 + 种子因子池
       ↓ 遗传算法变异/交叉 + AI主题生成
候选因子列表 (约120个/代)
       ↓ TemplateValidator 语法过滤
       ↓ CorrelationTracker 相关性过滤
通过验证的候选因子
       ↓ SimulatorTester 批量提交 WQ Brain
Sharpe/Fitness 回测结果
       ↓ BacktestStorage 存入 SQLite
Sharpe > 1.25 → discovered_alphas.txt 记录
Sharpe > 1.5  → 下一代初始种子池
```

## 3. 核心入口文件详解

### `continuous_evolution.py` — 主控制器（⭐ 最重要）

**作用**：整个系统的指挥中枢，无限循环挖掘因子。

**内置组件（今日集成）**：

```python
# ① 持久化记忆：从数据库加载历史精英种子
historical_seeds = _load_historical_seeds(storage, min_sharpe=1.25)

# ② 动态字段加载：从2663个字段缓存加载（而非手写20个）
wq_fields = _load_wq_fields_from_cache("USA", 1, "TOP3000")
# → 变异/交叉的"原材料"从24个扩大到2664个

# ③ 主题驱动AI生成（12个研究主题轮换）
themes = ["波动率曲面", "盈利质量", "分析师修正", "社交媒体情绪", ...]
# → 防止AI每次生成同质化因子（解决SELF_CORRELATION）

# ④ 相关性预过滤
correlation_tracker.add_alpha(expr)
# → 提交前过滤avg_corr > 0.5的高相关因子

# ⑤ 发现日志：优质因子自动写入文件
_log_discovery(expr, sharpe, fitness, alpha_id, source="AI-Ollama")
# → 查看 discovered_alphas.txt 即可看到所有好因子
```

**种子池（knowledge_pool）七大类**：
- A. 基本面经典因子（Operating Income, FFO/Debt）
- B. 杠杆与资产密度（Fair Value Liabilities, EBIT）
- C. 期权隐含波动率
- D. 社交媒体情绪（scl12_buzz, 热度×收益反转）
- E. 价格/成交量微观结构（VWAP-Close, 动量反转）
- F. 外汇管理能力（FX Translation Gain）
- G. 盈利质量复合（EBITDA长线增长）

**遗传算法核心函数**：
- `smart_mutate(expr)` — 字段替换、算子替换、参数微调
- `crossover(a, b)` — 两个因子表达式结构杂交
- `inject_neutralization(expr)` — 自动注入 `group_neutralize()`
- `generate_ai_alphas(theme, n)` — 调用 Ollama 生成主题因子

---

### `build.py` — 系统构建器（22KB）

**作用**：自动化构建整个 generation_two 系统，包含：
- 检查所有依赖是否就绪

- 验证 API 连接

## 4. 各模块目录详解

### `core/` — 核心引擎

| 文件 | 大小 | 作用 |
|---|---|---|
| `simulator_tester.py` | 28KB | **⭐ 核心**：向 WQ Brain API 提交因子、等待回测完成、解析 Sharpe/Fitness 结果 |
| `template_validator.py` | **96KB** | **⭐ 核心**：AST 语法检验器，验证因子表达式合法性，防止提交无效因子 |
| `template_generator.py` | 48KB | 模板生成器，管理 datafield 和 operator 的组合逻辑 |
| `credential_manager.py` | 12KB | 认证管理，从 `credential.txt` 读取账号信息，维持 WQ Brain 登录 session |
| `fast_expr_ast.py` | 40KB | 快速 AST（抽象语法树）解析器，支撑 template_validator |
| `expression_compiler.py` | 17KB | 因子表达式编译器，规范化表达式格式 |
| `region_config.py` | 4KB | 各地区（USA/EUR/CHN等）配置，包括 universe 和 delay 设置 |
| `slot_manager.py` | 10KB | WQ Brain 提交槽位管理，避免超限 |
| `simulation_counter.py` | 5KB | 统计每日模拟次数，防止超额使用 |

#### `core/mining/` — 挖掘专用工具

| 文件 | 作用 |
|---|---|
| `correlation_tracker.py` | **⭐ 今日集成**：追踪因子间相关性，预过滤高相关候选，解决 SELF_CORRELATION 失败 |
| `duplicate_detector.py` | 去重检测，防止完全相同因子重复提交 |
| `mining_coordinator.py` | 多进程挖掘协调器（未来可并行化） |
| `search_strategy.py` | 搜索策略（贪心/随机/定向） |

---

### `ollama/` — 本地 AI 接口层

| 文件 | 大小 | 作用 |
|---|---|---|
| `ollama_manager.py` | **78KB** | **⭐ 核心**：管理本地 Ollama 大模型，发送 prompt、解析响应、提取因子表达式 |
| `region_theme_manager.py` | 8KB | 管理各地区的研究主题库，驱动主题轮换逻辑 |
| `ollama_health.py` | 4KB | 检查 Ollama 服务是否运行正常（http://localhost:11434） |
| `start_ollama.py` | 12KB | 自动启动 Ollama 服务 |
| `check_ollama_models.py` | 6KB | 列出已下载的本地模型 |
| `pull_qwen_model.py` | 3KB | 自动下载 qwen2.5-coder 模型 |
| `duplicate_detector.py` | 11KB | AI生成因子的高级去重 |

**当前使用模型**：
- `qwen2.5-coder:1.5b` — 代码专用模型（当前，1.5GB）
- `llama3.2:latest` — 通用模型备用（2.0GB）
- 升级推荐：`qwen2.5-coder:7b`（7GB，更强逻辑推理）

## 8. 运行流程 Quick Start

### 前置条件

```bash
# 1. 安装依赖
pip install -r generation_two/requirements.txt

# 2. 确保 Ollama 已安装并在后台运行
# 下载：https://ollama.com
ollama pull qwen2.5-coder:1.5b   # 代码专用模型

# 3. 配置账号（generation_two/credential.txt）
email:你的WorldQuant账号@example.com
password:你的密码
```

### 首次安装：爬取全量数据字段

```bash
cd worldquant-miner
python refresh_data_fields.py
# 约 3 分钟，获取 2663 个字段
```

### 启动挖掘

```bash
# 主程序（推荐，无限运行）
$env:PYTHONIOENCODING="utf-8"
python -m generation_two.continuous_evolution

# 可视化 GUI（可选，并行开启）
python generation_two/gui/run_gui.py
```

### 查看成果

```bash
# 查看所有高质量因子（Sharpe > 1.25）
cat generation_two/discovered_alphas.txt

# 查看数据库统计
python generation_two/check_db.py

# 列出所有可用字段和算子
```
